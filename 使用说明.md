# NSEP 网站使用说明

## 一、日常使用方法

### 方法 1：双击启动（推荐）

直接双击项目根目录下的 **`启动网站.bat`** 文件即可！

### 方法 2：命令行启动

在项目目录下打开终端，运行：
```powershell
mkdocs serve
```

然后在浏览器打开：http://127.0.0.1:8000/

---

## 二、部署到 GitHub Pages

### 步骤 1：创建 GitHub 仓库

1. 访问 https://github.com
2. 点击 "New repository" 创建新仓库
3. 仓库名称填写：`nsep`
4. 选择 Public 或 Private（Private 需要付费才能使用 GitHub Pages）
5. **不要**勾选 "Initialize this repository"
6. 点击 "Create repository"

### 步骤 2：初始化 Git 仓库

在项目目录下打开终端，依次运行：

```powershell
# 初始化 Git
git init

# 添加所有文件
git add .

# 首次提交
git commit -m "Initial commit"

# 关联 GitHub 仓库（将 你的用户名 替换为你的 GitHub 用户名）
git remote add origin https://github.com/你的用户名/nsep.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

### 步骤 3：启用 GitHub Pages

1. 打开你的 GitHub 仓库页面
2. 点击 **Settings**（设置）
3. 左侧菜单找到 **Pages**
4. 在 **Build and deployment** 部分：
   - Source 选择：`Deploy from a branch`
   - Branch 选择：`gh-pages` 分支（后续部署后会自动创建）
   - 文件夹选择：`/ (root)`
5. 点击 **Save**

### 步骤 4：部署网站

在项目目录下运行：

```powershell
# 安装 ghp-import（首次部署需要）
pip install ghp-import

# 构建并部署到 GitHub Pages
mkdocs gh-deploy
```

完成后，你的网站将可以通过以下地址访问：
```
https://你的用户名.github.io/nsep/
```

### 步骤 5：更新网站内容

每次修改内容后，只需：

```powershell
# 1. 提交代码到 main 分支
git add .
git commit -m "更新内容"
git push

# 2. 重新部署
mkdocs gh-deploy
```

## 三、常用命令

| 命令 | 说明 |
|------|------|
| `mkdocs serve` | 启动本地开发服务器 |
| `mkdocs build` | 构建静态网站（输出到 site/ 目录） |
| `mkdocs gh-deploy` | 部署到 GitHub Pages |

## 五、添加新页面

1. 在 `docs/` 对应的目录下创建新的 `.md` 文件
2. 编辑 `mkdocs.yml` 中的 `nav` 部分，添加新页面的导航
3. 保存后，本地服务器会自动刷新

示例：在专业基础课下添加新页面

```yaml
nav:
  - 专业基础课:
    - 专业基础课: professional-basic/index.md
    - 高等数学: professional-basic/math.md
```
