## 经验之谈

### 笔蔓越莓莓（24-25夏）

> **[查看原帖](https://www.cc98.org/topic/6229120/1#6)**

机械学院的嵌入式系统没有作业，考试（闭卷）占比60%，点名占比10%，实验课占比30%，其中实验课点名占20%，实验报告占20%，实验考（夏七周）占60%。理论部分林勇刚老师来讲，实验部分由王郑拓、李宏娟、林志伟三位老师负责。

林勇刚老师人是很好的，但是没上实验课的时候理论真的很难理解，导致做实验的时候我也是一头雾水，做完实验弄懂实验后再看理论，又是恍然大悟，仿佛这本教材有防自学机制，只有你懂了知识才能看懂。

嵌入式系统需要C语言基础，理论考范围是嵌入式系统简介、STM32单片机结构和最小系统、STM32软件开发基础、通用输入输出GPIO、外部中断EXTI、通用同步/异步通信USART、定时器TIME、直接内存访问DMA。其中DMA实验考是不考的，结果理论期末考的大题考了，还是程序填空题，让所有人都汗流浃背了。由于我们这届是第二届改闭卷考的，只有一份历年卷，很多考点在复习PPT的时候都没能注意到，我相信随着回忆卷的增多，学习难度会逐渐降低的。

考试题型包括选择题、判断题、简答题、程序题。简答题很多题目是上一届考试的选择判断题，不知道下一届会不会这样。PPT看一遍是远远不够的，要多看，多背。我们的程序题目与往年不一样，没有选择了，不需要手写主循环里的函数了，而是直接全部是程序填空！DMA的程序题除了程序填空外还让我们写几行代码的注释。其中库函数和结构体里的内容都会在附录提供，但是库函数后边跟的是什么，全都要靠自己写，而这些都是实验课时IDE自动补全的部分，所以我建议整个代码都要自己过一遍，比如说 `GPIO_ InitStructure.GPIOSpeed = GPIOSpeed_50MHz` 的MHz的大小写我就纠结了很久（结果发现后面的程序也有这一行代码可参考，不用纠结了）。

实验课从夏四周开始上，三次课，一次实验考。实验考开卷，可翻实验指导书和教材，但不准在书中夹带纸张，不准插U盘。满分60分，第一道40分较基础，主要是GPIO；第二道20分比较难，会考中断和串口通讯，甚至定时器。由于大家实验时间是分布在周一至周五的，考虑到试题可能会被分享，实验考难度会从周一到周五逐渐递增。我是周二下午考的，感觉较简单，周五的同学比较难且给分严格，比如说第二题就考到了定时器。不过我实验考也没完全做出来，第一题有一个按下按键翻转一次LED的状态，我要么按键不灵，要么长按翻转了不止一次；第二题按下按键向串口发送字符串，题目是按键按下触发，我实现的结果是松开触发（怪我粗心了），扣了不少分。

### 明玲玉珑（24-25夏）

> **[查看原帖](https://www.cc98.org/topic/6228452/2#4)**

**考核**（姚鑫骅老师）：会有数字签到。

这门课分为实验课和理论课两部分，各有各的考试。

实验课有四次，前三次都是做实验，最后要整理所有实验，提交一份实验报告PDF。最后一次课考试，两道题，一道比较基础（占40分，主要是GPIO初始化、主函数逻辑），一道比较难（占20分，会考到中断优先级、串口通讯），按得分点给分。操作寄存器的程序写法不要求掌握。

理论课没有平时作业，就最后一个闭卷考试。内容涉及微机原理、嵌入式软硬件基础、实验课所有内容和DMA。题型有选择、判断、简答、编程。我们应该是改成闭卷考试的第二届，相比于上一届，我们没有“在下列十个英文缩写中选五个，写出其中英文全称”这种题，另外，编程题全部是填空，没有自行从零编写整个函数的题。我印象很深刻的是主客观题都考了DMA（因为我根本没复习）。

**上课体验**：我是一节课没听，选择跟着b站UP主“江协科技”的视频自学。他讲得很好，而且给了源码，直接爽抄。但是后面松懈了，串口和DMA没学，考试（both理论and实验）的时候直接送人头了。

朋友两天内刷完了智云并且做了超级详细的全套笔记，我跟着沾光，虽然没背下来多少。据她所说，姚鑫骅上课还蛮不错的，推荐跟着听下来（狂扇课上玩手机的自己的耳光）。

**应试**：最后有复习课，可以先看一下心里有个数。复习的时候最好事无巨细，该背的是没法偷懒的，毕竟简答题一个6分，背不下来的话基本上是分数大甩卖了。编程题最好是自己动手写，有助于发现很多细节上的问题。考试的时候会给函数框架，比如

```c
void GPIO_Init(GPIO_TypeDef* GPIOx, GPIO_InitTypeDef* GPIO_InitStruct)
void GPIO_WriteBit ( GPIO_TypeDef* GPIOx, u16 GPIO_Pin, BitAction BitVal)
typedef struct 
{ 
u16 GPIO_Pin; 
GPIOSpeed_TypeDef GPIO_Speed; 
GPIOMode_TypeDef GPIO_Mode; 
}  GPIO_InitTypeDef; 
```

给的挺多的，这个不用担心，着重记一下具体是多少参数、什么模式，注意拼写。编程题有些空还能前后对着抄，有点考政治的时候大题抄选择题的意思。

> **[查看原帖](https://www.cc98.org/topic/6129809)**

这门课主要是讲stm32的入门的，理论内容比较简单，偏向记忆（闭卷考，考试有一个题是让你写英文缩写的，不是哥们）。比较有压力的是实验和实验考，实验需要在规定时间内完成指定内容，lz stm32小白做的有点吃力。实验考是两道大题目，每道大题目下面有几个小题目，有送分的也有稍微难一点的。总的来说，拿分不难，拿满分需要谨慎（因为只有一次验收机会，所以一定要把题目读清楚，以免因为没理解清楚题意而丢分！）。

实验课考试题回忆：

**一、按键输入实验**

实现下面三个要求：

1. 按下KEY0则led0亮，松开KEY0则led0灭；
2. 按下KEY1则led1亮，松开KEY1则led1灭；
3. 按下KEY_UP则蜂鸣器响，松开KEY_UP则蜂鸣器关闭。

**二、中断与串口通讯**

要求：

1. 主函数中设置LED0 200ms交替闪烁；
2. 按下KEY0，LED0和LED1同时闪烁10次，10次后蜂鸣器响2s；
3. 按下KEY_UP，向上位机发送一条任意字符串（悄咪咪地说直接用printf打印就行了，不用那个函数UART_SendData，我一开始用这个函数只能打印一个字符）；
4. 设置KEY_UP的优先级高于KEY0。
